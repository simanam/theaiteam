# theaiteam — Claude Code Project Memory

This directory is **The AI Team**: a reusable, provider-agnostic AI workforce that takes a project brief and delivers code, plans, docs, designs, and marketing assets end-to-end.

## What this is

A workspace containing:
- **12 role agents** in [team/roles/](team/roles/) — PM, Architect, Backend, Frontend, Mobile, Designer, QA, Security, DevOps, Tech Writer, Marketing Strategist, **Team Maintainer** (the meta-role that keeps theaiteam itself healthy)
- **Skills library** in [team/skills/](team/skills/) — reusable workflow modules organized by domain
- **Rules** in [team/rules/](team/rules/) — always-follow guidelines (coding style, git, testing, security, docs, team-improvement)
- **Commands & workflows** in [team/commands/](team/commands/) and [team/workflows/](team/workflows/) — slash entries and multi-role pipelines
- **Orchestrator** in [orchestrator/](orchestrator/) — routes briefs to roles, manages task board, coordinates the shared `workspace/`
- **Adapters** in [adapters/](adapters/) — sync the provider-neutral `team/` source-of-truth into Claude Code, Codex, or Gemini formats
- **Briefs** in [briefs/](briefs/) — drop a project brief here to start a new project
- **Improvement briefs** in [improvement-briefs/](improvement-briefs/) — proposals to fix theaiteam itself (Maintainer-led)
- **Workspace** in [workspace/](workspace/) — per-project shared scratchpad, task board, and artifacts
- **Gap log** at [_team-gaps.md](_team-gaps.md) — running queue of identified gaps in theaiteam; any role appends, Maintainer drains

## How to use

1. Copy [briefs/_template.md](briefs/_template.md) to `briefs/<project-slug>.md` and fill it out
2. Run the orchestrator: invoke `/kickoff <project-slug>` (or read [orchestrator/orchestrator.md](orchestrator/orchestrator.md) and follow the protocol manually)
3. The orchestrator creates `workspace/<project-slug>/` with task board + artifact dirs and dispatches roles in the right order

## Self-improvement

The team detects and fixes its own gaps:

- **Any role** can call `/log-gap <severity> <title>` mid-task when it spots a defect (stale rule, broken link, missing skill). The gap goes into [_team-gaps.md](_team-gaps.md).
- **`/audit`** runs a read-only sweep (consistency, frontmatter, coverage, staleness, upstream drift, adapter drift) and populates the gap log.
- **`/self-improve`** drains the gap log: trivial issues auto-edited and committed; non-trivial filed as briefs in [improvement-briefs/](improvement-briefs/).
- The **team-maintainer** role is the only role that edits files under `team/` / `orchestrator/` / `adapters/`. Every other role logs and lets the maintainer fix.

## Key conventions

- **Source of truth lives in [team/](team/)** as plain markdown. Never hand-edit `.claude/`, `.codex/`, or `.gemini/` directly — those are generated by adapters.
- **Plans go in `docs/plans/<descriptive-name>.md`** per the user's global instructions.
- **Per-project plans go in `workspace/<project-slug>/plans/`** so they don't pollute the team-level repo.
- **Frontmatter is YAML**, structured per [team/rules/common/conventions.md](team/rules/common/conventions.md).
- **Roles delegate via the orchestrator**, not directly to each other, unless explicitly handed off via [team/commands/handoff.md](team/commands/handoff.md).

## Stack and tools

- **repomix** for ingesting target repos into AI-readable bundles ([tools/ingest.sh](tools/ingest.sh))
- **Playwright** for QA web testing
- **CodeQL / Semgrep** for security static analysis
- Model-agnostic — runs under Claude Code (primary), Codex CLI, Gemini CLI
